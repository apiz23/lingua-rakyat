Product Description

Lingua Rakyat is a multilingual civic AI assistant that enables Malaysian citizens to upload official government PDFs and ask questions in Bahasa Melayu, English, or Chinese. Powered by Retrieval-Augmented Generation (RAG), the system retrieves relevant document passages and generates source-cited, streamed answers in real time.

The platform serves citizens navigating documents from LHDN (tax), KWSP (pension), JPN (identity), and JPJ (road transport) — eliminating the need to visit government counters for basic queries. Every answer includes a faithfulness score measuring how grounded the response is in the source document.

FastAPI + SlowAPI Cohere Multilingual Embed v3 Pinecone Vector DB Groq LLaMA 3.3 70B Next.js 16 Voice I/O (Whisper + TTS)
Problem Statement / Research Gap
  • Malaysian government documents are dense, formal, and inaccessible to ordinary citizens regardless of education level
  • No existing AI tool provides native multilingual support in Malay, English, and Chinese over official government PDFs
  • Current AI assistants are not document-grounded — answers cannot be verified against source material, risking misinformation
  • Citizens must physically attend counters or call hotlines for basic queries, burdening both citizens and agencies
Research Objective
  • Design a multilingual RAG pipeline supporting Bahasa Melayu, English, and Chinese over government PDFs
  • Deliver source-cited answers with retrieval confidence and faithfulness scoring for verifiable citizen decision-making
  • Prototype a production-ready civic AI platform deployable by Malaysian government agencies at scale
  • Validate retrieval accuracy through a built-in ROUGE/BLEU evaluation and benchmarking module
Methodology / Process
1
Problem Analysis & Literature Review
Surveyed existing GovTech portals and AI assistants; identified multilingual grounding gap. Reviewed RAG architectures, embedding models, and reranking literature.
2
Architecture Design & Tech Selection
Selected Cohere multilingual embeddings for cross-lingual parity, Pinecone for vector storage, Groq for low-latency inference. Designed section-aware chunking (360w / 45w overlap) to preserve document structure.
3
RAG Pipeline & API Development
Built ingestion → multi-query augmentation (×4 variants) → reranking → evidence guard → streamed generation pipeline. Implemented FastAPI with per-IP rate limiting (SlowAPI) and BOOTH_MODE for demo events.
4
Frontend & Accessibility Design
Built Next.js 16 civic UI using Bricolage Grotesque (headings) and Atkinson Hyperlegible (body, accessibility font). Integrated Voice I/O (Groq Whisper STT + ElevenLabs TTS), PDF viewer with citation linking, multilingual EN/MS/ZH toggle.
5
Evaluation & Benchmarking
Built evaluation module computing ROUGE-1/2/L, BLEU, Flesch-Kincaid readability, and Cohere faithfulness score per answer. Annotated test suite for MyKad and Passport documents; streamed benchmark results in real time.

Usefulness / Potential Impact
  • Reduces counter queues — citizens get instant, source-backed answers without visiting LHDN, KWSP, or JPN offices
  • Language-inclusive — serves Malay, English, and Chinese speakers natively at the embedding level, not via post-hoc translation
  • Verifiable by design — every answer shows the exact source passage, page number, and a faithfulness score; citizens can verify accuracy
  • Agency-scalable — any Malaysian government agency can deploy by uploading their own PDF corpus; no model retraining required
  • Accessible — Atkinson Hyperlegible typeface, voice I/O, and high-contrast civic UI support low-vision and older users
