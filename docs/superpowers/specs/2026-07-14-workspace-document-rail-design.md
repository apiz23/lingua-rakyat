# Workspace Document Rail — Design

## Problem

`app/(app)/workspace/page.tsx` fetches the document list on mount but never
renders it (`setDocuments` result is discarded — `const [, setDocuments] =
useState<Document[]>([])`). It silently auto-anchors on the first ready doc.
Citizens land in the multi-doc chat with no way to see what's actually been
uploaded/available, and the empty-state suggested questions (`EmptyState`
chips) are hardcoded generic examples (LHDN e-Filing deadline, tax relief for
parents) that don't match the real seeded library (JPN MyKad, Imigresen
passport, KWSP e-Caruman, PTPTN myWaqaf) — clicking a chip can trigger an
"insufficient evidence" answer because that agency's document doesn't exist.

Two problems, one root: users have no visibility into what's askable.

## Goals

- Make the document library visible inside the workspace chat view.
- Suggested questions should reflect documents that actually exist.
- No regression to `/manage` (`doc-panel.tsx`) or existing multi-doc /
  `@`-mention behavior.

## Non-goals

- Redesigning `/manage` itself.
- Changing retrieval/RAG behavior (separate fix already shipped this session
  in `rag_pipeline.py`).
- Per-document suggested questions generated by the backend/LLM — out of
  scope for this pass; agency-keyed static map is enough.

## Architecture

### `frontend/hooks/useDocuments.ts` (new)

Single fetch point: `{ documents, readyDocs, loading, reload }`. Consumed by
`WorkspacePage`, `WorkspaceDocRail`, and optionally `ChatPanel` (new optional
props `documents?`/`onReload?` — when absent, `ChatPanel` keeps its existing
self-fetch so `/manage`'s standalone usage and any other caller is
unaffected). `doc-panel.tsx` is untouched — it stays self-contained.

### `frontend/components/workspace-doc-rail.tsx` (new)

Collapsible left rail rendered between `AppSidebar` and the chat column in
`workspace/page.tsx`. Reuses the list-item visual pattern from
`doc-panel.tsx` (status icon/badge, featured vs. user docs split, search) but
trimmed — no duplicate branding/back/benchmark header since `AppSidebar`
already has those. Compact header: collapse toggle, search, upload button.

Collapse state: `useLocalStorageSetting("lr-doc-rail-open", true)` — visible
by default. Hidden below `md:` breakpoint (mobile keeps the existing floating
upload button pattern; no mobile rail in this pass).

Clicking a doc → `setSelectedDoc` (anchor), unchanged semantics: doesn't
restrict multi-doc scope unless "ask all" is toggled off elsewhere.

### `frontend/lib/agency-questions.ts` (new)

`AGENCY_QUESTION_MAP`: keyed by the real seeded agencies (`JPN`,
`IMIGRESEN`, `KWSP`, `PTPTN`), 1-2 questions each per language (ms/en/zh),
written against what's actually in those docs (MyKad renewal, passport
requirements, e-Caruman/KWSP password reset, myWaqafPTPTN eligibility) —
replaces the current fixed `CHIPS` map in `empty-state.tsx`.

### `frontend/components/chat-panel/empty-state.tsx` (modified)

Takes new `readyDocs: Document[]` prop. Chip list built as:

1. One question per distinct agency present among `readyDocs` (cap 4).
2. If room and any non-featured (user-uploaded, no `agency`) ready docs
   exist and there are ≤2 of them, add one generic per-doc chip: "What does
   {docname} cover?" (localized).
3. If room remains, add one generic "ask anything about your documents"
   chip.

States:
- `readyDocs` loading → skeleton chips, not stale generic ones.
- `readyDocs.length === 0` → single chip prompting upload, no fake
  questions about documents that don't exist.

## Data flow

```
workspace/page.tsx
  useDocuments() -> { documents, readyDocs, loading, reload }
    -> WorkspaceDocRail (documents, selectedDoc, onSelectDoc, onReload)
    -> ChatPanel (selectedDoc, documents, readyDocs via props)
         -> EmptyState (readyDocs) [only when no selectedDoc/messages]
```

## Edge cases

- Rail collapsed + zero docs: collapse toggle still reachable, not stuck
  hidden.
- Search in rail, zero results: reuse existing empty-search copy pattern
  from `doc-panel.tsx`.
- Mobile (`<md:`): rail hidden, existing floating upload button remains the
  only mobile entry point (unchanged from today).

## Testing

Manual verification only (no dedicated test framework for this UI layer
currently):
- Rail shows on load, collapses/expands, persists across reload.
- Clicking a doc in rail sets it as chat anchor; existing chat/session logic
  unaffected.
- Chips reflect actually-seeded agencies; clicking a chip produces a real
  answer (not `evidence_mode: "insufficient"`).
- `/manage` page (`doc-panel.tsx`) unaffected — spot check after changes.
